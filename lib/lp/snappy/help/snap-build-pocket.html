Pocket for snap builds

Each distribution series in a package repository is divided into several parts, called pockets. These refer to different update streams that users of that series may wish to use. You can similarly choose to build snap packages based on any of those different update streams.

If unsure, choose Updates.

Release
The package versions that were published when the distribution release was made.
Security
Package versions containing security fixes for the released distribution. Packages in Security are built with dependencies from Release and Security.
Updates
Updates for serious bugs that do not affect the security of the system. Packages in Updates are copied from Proposed or Security once they have been tested.
Proposed
The testing area for updates. This repository is recommended only to those interested in helping to test updates and provide feedback. Packages in Proposed are built with dependencies from Release, Security, Updates and Proposed.
Backports
As the name states, these are unsupported new versions of packages which have been backported to an older release. Packages may contain new features, may introduce new interfaces, and bugs. Packages in Backports are built with dependencies from Release, Security, Updates and Backports.
