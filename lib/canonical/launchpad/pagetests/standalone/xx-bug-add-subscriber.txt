Non-Personal Subscriptions
==========================

In addition to subscribing / unsubscribing yourself, it is possible to add a
person as a subscriber to the bug. You would use this feature to subscribe
someone else to the bug.

First, let's make sure we can see the page.

  >>> print http(r"""
  ... GET /products/firefox/+bug/1/+addsubscriber HTTP/1.1
  ... Authorization: Basic Zm9vLmJhckBjYW5vbmljYWwuY29tOnRlc3Q=
  ... """)
  HTTP/1.1 200 Ok
  ...

Now, let's POST to it, to subscribe sabdfl.

  >>> print http(r"""
  ... POST /products/firefox/+bug/1/+addsubscriber HTTP/1.1
  ... Authorization: Basic Zm9vLmJhckBjYW5vbmljYWwuY29tOnRlc3Q=
  ... Content-Length: 306
  ... Content-Type: multipart/form-data; boundary=---------------------------12077743959310110181447415077
  ... 
  ... -----------------------------12077743959310110181447415077
  ... Content-Disposition: form-data; name="field.person"
  ... 
  ... sabdfl
  ... -----------------------------12077743959310110181447415077
  ... Content-Disposition: form-data; name="UPDATE_SUBMIT"
  ... 
  ... Add
  ... -----------------------------12077743959310110181447415077--
  ... """)
  HTTP/1.1 303 See Other
  ...
  Location: http://.../products/firefox/+bug/1
  ...

Now, we should see Mark Shuttleworth as a subscriber.


  >>> print http(r"""
  ... GET /products/firefox/+bug/1 HTTP/1.1
  ... """)
  HTTP/1.1 200 Ok
  ...Mark Shuttleworth...
