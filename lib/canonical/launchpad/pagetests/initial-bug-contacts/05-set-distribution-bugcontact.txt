To set a bug contact for a distribution, we need to go to the Bug page
of that distribution:

  >>> browser.open('http://launchpad.dev/ubuntu/+bugs')

But the link is not available if you are not logged in with permission
to change the bug contact.

  >>> browser.getLink("Change bug contact")
  Traceback (most recent call last):
  ...
  LinkNotFoundError

Now we login as Mark...

  >>> browser.addHeader('Authorization', 'Basic mark@hbd.com:test')
  >>> browser.reload()
  >>> browser.url
  'http://launchpad.dev/ubuntu/+bugs'

...and we can see that the link is there.

  >>> bugcontact_link = browser.getLink("Change bug contact")
  >>> bugcontact_link.url
  'http://launchpad.dev/ubuntu/+bugcontact'

Anyone with launchpad.Edit permission can edit the distribution bug
contact. In this example, we're using mark@hbd.com to set the
distribution bug contact.

  >>> bugcontact_link.click()
  >>> browser.url
  'http://launchpad.dev/ubuntu/+bugcontact'

The bug contact page takes just one simple value: the bug contact email
or nickname. Let's set carlos@canonical.com as the bug contact for
Ubuntu.

  >>> browser.getControl("Bug Contact").value = 'carlos@canonical.com'
  >>> browser.getControl("Change").click()

And then we're redirected to the distribution page.

  >>> from canonical.launchpad.ftests.test_pages import find_tags_by_class

  >>> browser.url
  'http://launchpad.dev/ubuntu'

  >>> for tag in find_tags_by_class(browser.contents, "informational message"):
  ...     print tag
  <div...Successfully changed the distribution bug contact to carlos@canonical.com...

Now that the bug contact is set, there's a link to him on the
distribution page.

  >>> browser.getLink(
  ...   url='/~carlos').url
  'http://launchpad.dev/~carlos'

When the bug contact doesn't have an email address, the displayname is
shown instead in the feedback message.

  >>> browser.open("http://launchpad.dev/ubuntu/+bugcontact")
  >>> browser.getControl("Bug Contact").value = 'landscape-developers'
  >>> browser.getControl("Change").click()

  >>> for tag in find_tags_by_class(browser.contents, "informational message"):
  ...     print tag
  <div...Successfully changed the distribution bug contact to Landscape Developers...
